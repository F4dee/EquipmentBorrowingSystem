Create a complete set of UI/UX wireframes for an Equipment Borrowing System web application. The system has two user roles: STUDENT and ADMIN.

STUDENT FACES:
1. Login Page
   - Email and password fields
   - "Login" button
   - "Register here" link
   - "Login with Google" button
   - Clean, centered layout

2. Registration Page
   - Full name field
   - Student ID field
   - CIT-U email field
   - Password field
   - Confirm password field
   - "Register" button
   - "Back to Login" link

3. Equipment Catalog Page
   - Header with logo, search bar, cart icon, user menu
   - Filter sidebar (categories: Router, Switch, VR Headset, etc.)
   - Main content: Grid view (3 columns) of equipment cards
   - Each card shows: equipment image, name, category, available quantity, "Add to Cart" button
   - Availability status badge (Available/Low Stock/Out of Stock)

4. Item Detail Page
   - Large equipment image
   - Item name and category
   - Description
   - Technical specifications
   - Availability calendar showing borrowed dates
   - Quantity selector (1-5)
   - Borrow date picker
   - Return date picker
   - "Add to Cart" button
   - "Back to Catalog" link

5. Borrowing Cart Page
   - List of items in cart
   - Each row shows: item image, name, quantity, borrow date, return date, remove icon
   - Order summary showing total items
   - "Purpose/Reason for borrowing" textarea
   - "Submit Request" button
   - "Continue Shopping" button

6. My Requests Page
   - Tabs for: Pending, Approved, Borrowed, Returned, All
   - List of requests with:
     - Request number
     - Borrow date range
     - Status badge (color-coded)
     - Items count
     - View details button
   - Clicking request expands to show items

7. Report Issue Page
   - Lab location dropdown
   - PC/Equipment number field
   - Problem description textarea
   - Image attachment button
   - "Submit Ticket" button

8. User Profile Page
   - Profile picture
   - Full name
   - Student ID
   - Email
   - Role badge
   - Borrowing statistics (total requests, active loans)
   - "Edit Profile" button

ADMIN FACES:
9. Admin Dashboard
   - Sidebar navigation: Dashboard, Requests, Inventory, Tickets, Users
   - Top stats cards: Pending Requests, Active Loans, Open Tickets, Total Items
   - Recent pending requests table
   - Recent tickets table
   - Quick actions section

10. Request Queue Page
    - Table of pending requests
    - Columns: Request #, Student, Borrow Date, Return Date, Items, Actions
    - Each row has "Approve" and "Deny" buttons
    - Click row to see full details
    - Filter by date range

11. Inventory Management Page
    - Table of all equipment
    - Columns: Image, Name, Category, Total Qty, Available Qty, Status, Actions
    - "Add New Item" button
    - Edit and Delete buttons per row
    - Search and filter options

12. Ticket Management Page
    - Tabs: Open, In Progress, Resolved
    - List of tickets with:
      - Ticket number
      - Lab location
      - Reporter name
      - Description preview
      - Status badge
      - "View" and "Update Status" buttons

13. User Management Page
    - Table of all users
    - Columns: Name, Student ID, Email, Role, Joined Date, Actions
    - Filter by role
    - Edit role dropdown

Style: Clean, modern, minimalist wireframes with Material Design principles. Use blue as primary color (#2563EB). Show proper spacing, alignment, and UI elements. Grayscale with blue accents. Include mobile-responsive view for key screens. Suitable for a university system design document.
